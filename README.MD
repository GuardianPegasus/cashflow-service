# Веб-сервис для управления движением денежных средств (ДДС)

Веб-приложение для ведения учета финансовых операций (поступлений и списаний) с жестким контролем логических связей между сущностями.

## Стек
* Python 3.12
* Django 5.x
* PostgreSQL 15
* HTML5 / Vanilla JavaScript / Bootstrap 5
* Docker

## Особенности
1. Создание, чтение, редактирование и удаление (CRUD) финансовых транзакций.
2. На стороне клиента JavaScript налету фильтрует доступные Категории в зависимости от выбранного Типа (Пополнение/Списание), а Подкатегории — в зависимости от выбранной Категории.
3. Метод `clean()` на уровне моделей Django дублирует проверку бизнес-логики, предотвращая появление некорректных связей в СУБД (например, привязку расхода к доходной категории).
4. Выделенный веб-интерфейс на Bootstrap-вкладках для управления Статусами, Типами, Категориями и Подкатегориями без необходимости заходить в админ-панель.

## Как запустить проект локально

1. Клонируйте репозиторий:
```bash
git clone https://github.com/GuardianPegasus/cashflow-service
cd cashflow-service
```
2. Запустите команду сборки:
```bash
docker-compose up --build
```
3. Примените миграции базы данных:
```bash
docker-compose exec web python manage.py migrate
```
4. Создайте суперадмина (опционально, для доступа к административной панели Django):
```bash
docker-compose exec web python manage.py createsuperuser
```
5. Откройте приложение:
* Перейдите в браузере по адресу: http://127.0.0.1:8000
* Админ-панель доступна по адресу: http://127.0.0.1:8000/admin/
